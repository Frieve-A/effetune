# 음악 라이브러리 사용 방법

버전 2.1.0에서는 카탈로그 스키마 v3를 사용하는 CUE 지원 음악 라이브러리가 도입되었습니다. 이전 음악 라이브러리에 등록한 폴더와 플레이리스트는 이 카탈로그로 이전되지 않으므로 음악 폴더를 다시 추가하고 플레이리스트도 새로 만들거나 다시 가져오세요. 이전 카탈로그와 오디오 파일은 변경되지 않습니다.

음악 라이브러리는 선택한 음악 폴더를 인덱싱하여 로컬 음악 컬렉션을 트랙, 앨범, 아티스트, 장르, 하위 폴더, 폴더, 최근 추가됨, 플레이리스트별로 찾아볼 수 있게 해 주는 기능입니다. 재생되는 소리는 일반 음악 파일 재생과 마찬가지로 현재 EffeTune의 Effect Pipeline을 통과합니다.

음악 라이브러리가 저장하는 것은 앱 안의 카탈로그, 아트워크 캐시, 플레이리스트입니다. 음악 파일 자체를 편집하거나, 이름을 바꾸거나, 이동하거나, 삭제하지 않습니다.

## 사용 가능한 환경

- **데스크톱 앱:** 모든 기능을 갖춘 폴더 스캐너를 사용하며, 선택한 폴더를 다음 실행 때도 사용할 수 있습니다. 데스크톱 버전에서는 트랙을 파일이 있는 폴더에서 표시할 수도 있습니다.
- **File System Access를 지원하는 PC용 Chromium 계열 브라우저:** 선택한 폴더 핸들을 영구 저장합니다. 접근을 허용하면 새로 고친 뒤에도 핸들을 재사용할 수 있지만, 브라우저가 권한을 다시 요청할 수 있습니다.
- **모바일 브라우저, Safari, Firefox 등 File System Access를 지원하지 않는 브라우저:** 선택한 `File` 객체를 현재 페이지 세션 동안만 보관합니다. 카탈로그는 저장되지만 새로 고친 뒤에는 파일을 다시 열 수 없습니다. 새로 고칠 때마다 폴더나 파일을 다시 선택하세요. EffeTune은 정규화된 상대 경로를 사용해 기존 카탈로그 항목에 다시 연결합니다.

음악 라이브러리는 MP3, WAV, OGG, FLAC, Opus, M4A, AAC, WebM, MP4 등 일반적인 미디어 파일 확장자를 인덱싱합니다. 또한 외부 CUE 시트를 사용해 같은 폴더의 WAV 또는 FLAC 앨범 파일을 개별 곡으로 나눌 수 있습니다. MP4 파일은 영상을 표시하지 않고 오디오 트랙만 재생합니다. MP4 내부의 오디오 코덱을 포함한 실제 재생 가능 여부는 브라우저와 OS의 디코딩 기능에도 따라 달라집니다.

## 음악 라이브러리 열기

- **PC 레이아웃:** 헤더의 **음악 라이브러리** 버튼을 클릭합니다.
- **모바일 레이아웃:** 화면 아래쪽 내비게이션의 **라이브러리** 탭을 엽니다.
- **데스크톱 앱:** **보기 > 음악 라이브러리** 또는 **Ctrl+L**(macOS에서는 **Command+L**)로도 열 수 있습니다.

이펙트 편집으로 돌아가려면 PC 레이아웃에서는 **Effect Pipeline** 버튼을 클릭하고, 모바일 레이아웃에서는 **이펙트** 탭으로 돌아갑니다. 데스크톱 앱에서는 **보기 > Effect Pipeline** 또는 **Ctrl+E**(macOS에서는 **Command+E**)도 사용할 수 있습니다.

시작할 때 음악 라이브러리를 먼저 표시하려면 **설정 > 구성...**을 열고 **시작 시 표시:**를 **음악 라이브러리**로 설정합니다. **음악 라이브러리** 옆의 목록에서 처음 표시할 항목을 **트랙**, **앨범**, **아티스트**, **장르**, **하위 폴더**, **폴더**, **플레이리스트** 중에서 선택할 수 있습니다.

## 음악 폴더 추가

1. 음악 라이브러리를 엽니다.
2. **음악 폴더 추가**를 선택합니다.
3. 음악 파일이 들어 있는 폴더를 선택합니다. 모바일이나 대체 방식을 쓰는 브라우저에서는 영구적인 폴더 접근 대신 폴더 안의 파일을 선택하는 화면이 열릴 수 있습니다.
4. 스캔이 끝날 때까지 기다립니다. 상태 줄에는 트랙 수와 앨범 수가 표시되며, 인덱싱 중에는 진행 상황도 표시됩니다.

이미 등록된 폴더 안에 있는 폴더를 추가하려고 하면 EffeTune은 중복으로 인덱싱하지 않고 경고를 표시합니다. 등록된 폴더를 포함하는 상위 폴더를 추가하는 경우에는 기존 폴더를 새 폴더로 통합할 수 있습니다.

## 탐색과 검색

내비게이션 탭으로 카탈로그를 전환합니다.

- **트랙** - 인덱싱된 모든 트랙을 표시합니다. PC 레이아웃에서는 정렬 가능한 표로, 모바일 레이아웃에서는 간결한 목록으로 표시됩니다.
- **앨범** - 메타데이터를 기준으로 앨범 단위로 묶습니다.
- **아티스트** - 메타데이터의 아티스트와 앨범 아티스트를 기준으로 묶습니다.
- **장르** - 메타데이터의 장르를 기준으로 묶습니다.
- **하위 폴더** - 각 트랙이 직접 들어 있는 폴더의 경로를 음악 폴더 루트에 대한 상대 경로로 묶습니다.
- **폴더** - 등록한 음악 폴더 루트와 스캔 상태, 실제 디렉터리 구조를 표시합니다.
- **최근 추가됨** - 최근 인덱싱된 트랙을 표시합니다.
- **플레이리스트** - 음악 라이브러리 안에서 만들었거나 가져온 플레이리스트를 표시합니다.

`Artist A; Artist B`처럼 세미콜론으로 구분된 앨범 아티스트 값은 전체 표기를 그대로 표시하면서 각 아티스트로 따로 인덱싱됩니다. `&`, `/`, `feat.`는 구분자로 처리하지 않습니다.

**폴더**에서 루트를 열면 **트리 보기**로 탐색할 수 있습니다. 트랙 목록 위에 하위 폴더가 표시되며, 폴더를 선택하면 그 안으로 이동합니다. 상단의 경로 표시나 **이전**을 사용하면 상위 단계로 돌아갈 수 있습니다. 트랙 목록에는 현재 단계에 직접 저장된 트랙만 표시됩니다. **평면 보기**로 전환하면 모든 하위 폴더를 포함하여 등록된 루트의 인덱싱된 트랙을 한꺼번에 표시합니다. 보기를 전환해도 현재 경로 위치는 유지되므로 **트리 보기**로 돌아왔을 때 같은 위치에서 탐색을 계속할 수 있습니다. **라이브러리 검색**은 라이브러리 전체를 대상으로 하므로 검색 중에는 하위 폴더 목록을 표시하지 않습니다.

폴더에 직접 저장된 트랙이 없고 하위 폴더가 정확히 하나만 있으면 **트리 보기**는 같은 조건이 연속되는 단계를 `Artist / Album`과 같은 한 행으로 합쳐 표시합니다. 이 행을 선택하면 가장 깊은 폴더로 바로 이동하며, 행에 표시되는 트랙 수도 해당 폴더를 기준으로 합니다. 상단의 경로 표시에서 중간 단계를 직접 열 수 있으며, 데스크톱의 **이전**은 한 번에 한 단계씩 돌아갑니다. 모바일의 **이전**은 직전에 보던 화면으로 돌아가므로, 합쳐진 경로의 중간 단계를 한 번에 건너뛸 수 있습니다.

**폴더** 트리는 실제 상대 경로를 정확히 구분하며, 메타데이터를 분석하는 중이거나 분석에 실패한 경우도 포함하여 발견된 모든 카탈로그 트랙 항목을 반영합니다. 폴더에 표시되는 수는 해당 폴더와 모든 하위 단계에 있는 트랙 항목의 합계입니다. CUE 트랙은 논리 트랙별로 계산하므로 여러 트랙이 하나의 WAV 또는 FLAC 원본 파일을 공유하더라도 물리적 오디오 파일 하나가 아니라 여러 트랙으로 계산됩니다. 반면 **하위 폴더**는 분석에 성공한 트랙을 정규화된 직접 상위 폴더 식별자로 묶은 메타데이터 기반 그룹입니다. 따라서 대소문자 또는 Unicode 정규화만 다른 경로는 하나의 그룹으로 합쳐질 수 있습니다.

예를 들어 `Artist/Album/01 Song.flac`은 `Artist/Album` 하위 폴더 그룹에 표시되고, 실제 **폴더** 트리에서는 `Artist`, `Album` 순으로 찾을 수 있습니다. 서로 다른 인덱싱 루트에 같은 상대 경로가 있어도 별도로 유지됩니다. 루트에 직접 저장된 파일은 하위 폴더 그룹을 만들지 않지만 **트랙**과 **폴더**에서 연 해당 루트의 직접 트랙 목록에서 계속 사용할 수 있습니다.

**라이브러리 검색**에서는 트랙, 앨범, 아티스트, 플레이리스트를 한꺼번에 검색할 수 있습니다. PC 레이아웃에서는 트랙 목록의 헤더에서 제목, 아티스트, 앨범, 장르, 시간 기준으로 정렬할 수 있습니다. 앨범, 아티스트, 장르, 하위 폴더, 플레이리스트 보기에는 카탈로그를 기준으로 동작하는 **정렬** 목록이 있습니다. 보기에 따라 이름, 아티스트, 연도, 경로, 트랙 수, 총 재생 시간, 업데이트 시간, 생성 시간을 오름차순 또는 내림차순으로 선택할 수 있습니다. 선택한 방식은 보기마다 따로 유지됩니다.

트랙 검색에서 3자 이상의 검색어는 제목, 아티스트, 앨범, 장르, 파일 이름, 경로의 어느 위치에서나 일치합니다. 1~2자 검색어는 단어의 시작 부분에만 일치합니다. 단어 중간을 검색하려면 3자 이상 입력하세요.

PC 및 모바일 레이아웃 모두에서 트랙 검색 결과, **트리 보기**의 현재 단계에 직접 저장된 트랙 또는 앨범, 아티스트, 장르, 하위 폴더, 재생목록 상세 결과가 300트랙 이하이면 모든 트랙이 기본적으로 선택됩니다. 301트랙 이상이면 자동으로 선택되지 않습니다. 행의 체크박스, **모두 선택**, **모두 선택 해제**로 선택 상태를 바꿀 수 있습니다.

모바일은 일반 제목 목록으로 시작하며 아티스트 열과 재생 시간 열은 표시하지 않습니다. 트랙을 길게 눌렀을 때만 선택 모드로 들어가 체크박스, **모두 선택**, **모두 선택 해제**가 나타나며 기존 행 작업도 계속 사용할 수 있습니다. 자동 선택, **모두 선택**, **모두 선택 해제**, 개별 체크박스 조작은 선택 상태만 변경하며 선택 모드를 시작하거나 종료하지 않습니다.

메타데이터가 없거나 읽을 수 없는 경우 EffeTune은 파일 이름과 폴더 정보를 사용해 표시합니다. 트랙 속성에서는 파일 경로, 형식, 샘플링 속도, 비트 깊이, 비트레이트, 주요 메타데이터 항목을 확인할 수 있습니다. CUE 트랙은 트랙 유형, CUE 파일 경로, 원본 오디오 경로, 원본 파일 안의 재생 구간도 표시합니다.

## CUE 앨범 파일

외부 `.cue` 파일을 그 안에서 지정한 WAV 또는 FLAC 파일과 같은 폴더에 둔 다음 해당 폴더를 추가하거나 다시 스캔하세요. 올바른 `TRACK ... AUDIO` 항목은 각각 음악 라이브러리의 개별 곡으로 표시됩니다. 제목, 연주자, 날짜, 장르, 트랙 번호는 CUE 정보가 있으면 이를 사용하고, 기술적인 오디오 정보는 원본 WAV 또는 FLAC에서 가져옵니다.

음악 라이브러리에 추가한 트랙은 먼저 원본 오디오 파일에 포함된 앨범 아트를 사용합니다. 포함된 이미지가 없으면 EffeTune은 CUE 파일과 같은 폴더에서 `cover.jpg`, `cover.png`, `front.jpg`, `front.png`, 원본 오디오 파일과 같은 이름의 JPEG 또는 PNG(오디오 확장자를 붙인 이름도 가능) 순으로 찾습니다. 데스크톱 앱에서 직접 재생할 때는 같은 인접 이미지 후보를 자동으로 사용하며, 이 재생 경로에서는 원본 오디오 파일에 포함된 이미지를 추출하지 않습니다. 브라우저에서 직접 재생할 때는 선택한 파일이나 등록된 폴더에서 접근할 수 있는 해당 이미지를 사용합니다.

CUE 앨범은 **Open music files** 로 바로 재생할 수 있으며, 모바일에서는 **Open Music** 을 선택합니다. 데스크톱 앱에서는 **File > Open music file...** 도 사용할 수 있고 `.cue` 파일만 선택하면 됩니다. PC용 Chromium 브라우저에서는 먼저 앨범 폴더를 음악 라이브러리에 추가하고 접근을 허용하세요. 그 뒤에는 `.cue` 파일만 선택해도 EffeTune이 등록된 폴더에서 참조된 WAV 또는 FLAC 파일과 해당 표지를 열며, 이 선택은 카탈로그에 추가되지 않습니다. File System Access를 지원하지 않는 브라우저에서는 여전히 `.cue` 파일 한 개와 그 파일이 참조하는 WAV 또는 FLAC 파일만 모두 함께 선택하고, 표지를 사용하려면 해당 이미지도 추가해야 합니다. 올바른 선택은 현재 재생 대기열을 바꿉니다. 확인에 실패하면 현재 대기열은 변경되지 않습니다.

CUE 시트가 잘못되었거나 원본 파일을 안전하게 찾을 수 없으면 EffeTune이 문제와 해결 방법을 안내하고 WAV 또는 FLAC을 파일 전체 길이의 일반 트랙으로 가져옵니다. CUE 시트나 파일 이름을 고친 뒤 폴더를 다시 스캔하세요.

## 라이브러리에서 재생

트랙, 앨범, 아티스트, 장르, 하위 폴더, 폴더, 검색 결과, 플레이리스트를 선택하고 다음 작업을 사용합니다.

- **재생** - 현재 플레이어 대기열을 바꾸고 재생합니다.
- **셔플** - 선택한 트랙 묶음을 무작위 순서로 재생합니다.
- **다음에 재생** - 선택한 트랙을 현재 트랙 다음에 삽입합니다.
- **대기열에 추가** - 선택한 트랙을 대기열 끝에 추가합니다.
- **플레이리스트에 추가** - 선택한 트랙을 음악 라이브러리의 플레이리스트에 저장합니다.

PC에서는 트랙 행을 더블 클릭해 그 위치부터 재생하거나, 마우스 오른쪽 버튼 또는 **더 보기** 메뉴에서 트랙 작업을 열 수 있습니다. 모바일에서는 일반 목록의 트랙을 탭해 재생합니다. 길게 누르면 위에서 설명한 선택 모드로 들어갑니다.

일반 음악 플레이어의 조작과 반복/셔플 설정은 그대로 사용할 수 있습니다. 키보드를 사용할 수 있는 기기에서는 일반 플레이어용 키보드 단축키도 사용할 수 있습니다. 폴더가 오프라인 상태가 되어 라이브러리 안의 트랙을 열 수 없다면 해당 폴더를 다시 연결하거나 다시 가져오세요.

## 폴더 업데이트와 다시 연결

음악 폴더 안의 파일을 추가, 삭제, 이름 변경, 태그 편집한 뒤에는 **다시 스캔**을 사용합니다. 다시 스캔하면 변경된 트랙이 업데이트되고, 찾을 수 없게 된 파일이 카탈로그에서 제외되며, 이전에 사용할 수 없었던 플레이리스트 항목도 다시 해결하려고 시도합니다.

**폴더** 화면의 상태 표시에서 해당 폴더를 사용할 수 있는지 확인할 수 있습니다.

- **OK** - 폴더를 사용할 수 있습니다.
- **스캔 안 됨** - 아직 인덱싱되지 않았습니다.
- **찾을 수 없음** - 폴더 또는 저장된 경로를 사용할 수 없습니다.
- **다시 연결** - EffeTune에 접근 권한을 다시 부여해야 합니다.

폴더에 **다시 연결**이 표시되면 **다시 연결**을 선택하고 같은 폴더에 대한 접근을 다시 허용합니다. 폴더를 제거해도 음악 라이브러리 카탈로그에서만 제외되며, 디스크의 파일은 삭제되지 않습니다.

## 플레이리스트

음악 라이브러리의 플레이리스트는 EffeTune 안에 저장되며, 인덱싱된 폴더 안의 트랙을 포함할 수 있습니다.

다음 작업을 할 수 있습니다.

- 선택한 라이브러리 트랙으로 플레이리스트 만들기.
- 현재 플레이어 대기열을 플레이리스트로 저장하기.
- 플레이리스트 이름 변경, 복제, 삭제, 순서 변경하기.
- 플레이리스트 안의 트랙을 드래그해 순서 바꾸기. 드래그하기 어려운 환경에서는 **위로 이동**, **아래로 이동**을 사용하기.
- M3U, M3U8, PLS, XSPF 형식의 플레이리스트 가져오기.
- 개별 플레이리스트를 연 뒤 **M3U8 내보내기** 또는 **XSPF 내보내기**로 내보내기.

### 최근 재생한 곡과 즐겨찾기

EffeTune은 두 개의 특수 플레이리스트를 일반 플레이리스트와 같은 카드 그리드에 나란히 표시합니다. 이 목록은 필요할 때만 만들어집니다. 인덱싱된 곡의 재생을 처음 시작하면 **최근 재생한 곡**이, 곡에 처음 별표를 지정하면 **즐겨찾기**가 만들어집니다.

- **최근 재생한 곡**에는 중복 없이 최근 100곡이 최신순으로 저장됩니다. 같은 곡을 다시 재생하면 그 곡이 맨 위로 이동합니다.
- **즐겨찾기**에는 ☆로 표시한 곡이 들어갑니다. PC에서는 곡 옆의 별표를 사용하고, 모바일에서는 곡의 **더보기** 메뉴를 여세요. PC에서 곡을 마우스 오른쪽 버튼으로 클릭해도 같은 메뉴를 열 수 있습니다.

이 플레이리스트의 이름은 고정되어 있으며 현재 UI 언어로 표시되므로 이름을 바꿀 수 없습니다. 일반 플레이리스트처럼 복제, 내보내기, 삭제는 할 수 있습니다. 삭제한 경우 다음 재생 기록이나 즐겨찾기 작업에 필요할 때 빈 목록으로 다시 만들어집니다. 카드의 아트워크 영역에는 시계 또는 별 아이콘이 표시되며, **즐겨찾기** 카드 오른쪽 아래의 재생 버튼을 누르면 바로 재생됩니다. 특수 플레이리스트는 일반 플레이리스트 검색 결과에는 포함되지 않습니다.

폴더를 스캔하면 트랙 인덱싱이 끝난 뒤 지원되는 플레이리스트 파일을 자동으로 가져오며, 내용이 바뀌지 않은 파일은 건너뜁니다. 같은 폴더와 상대 경로에 있는 파일의 내용이 바뀌면 자동으로 가져온 플레이리스트의 항목을 한 번에 교체합니다. 이때 EffeTune에서 해당 플레이리스트 항목에 적용한 편집도 교체됩니다. 가져오기에 실패했거나 취소한 경우 다음 재스캔에서 다시 시도합니다. 원본 파일을 삭제하거나 이름을 바꾸어도 기존 플레이리스트는 남으며, 이름을 바꾼 파일은 새 플레이리스트로 가져옵니다.

가져올 때는 현재 라이브러리 안의 트랙과 일치하는 항목 수가 미리 표시됩니다. 일치하지 않은 항목도 가능한 범위에서 연결 안 된 항목으로 보존되므로, 해당 폴더를 나중에 추가하거나 다시 연결하면 해결될 수 있습니다.

내보낼 때 **상대 경로**를 선택하면 가능한 경우 내보내기 대상 위치를 기준으로 한 상대 경로로 기록합니다. 음악 폴더와 함께 플레이리스트를 옮기고 싶을 때 유용합니다. M3U8과 XSPF는 앨범 파일 안의 CUE 트랙 재생 구간을 보존할 수 없으므로 EffeTune은 해당 트랙을 제외하고 그 곡 수를 알려 줍니다. 제외된 CUE 트랙 대신 앨범 파일의 실제 경로를 기록하지는 않습니다.

## 안전성과 저장 위치

- 음악 라이브러리는 음악 파일과 메타데이터를 읽지만, 음악 파일에 변경 사항을 쓰지 않습니다.
- 아트워크 캐시와 플레이리스트는 앱 안의 데이터이며, 음악 파일에 삽입되는 변경 사항이 아닙니다.
- 하위 폴더 그룹은 카탈로그에 이미 저장된 상대 경로에서 만들어집니다.
- 브라우저 저장소는 브라우저 설정이나 사용자 조작으로 삭제될 수 있습니다. 중요한 플레이리스트는 필요에 따라 내보내세요.
- File System Access를 지원하는 브라우저에서는 새로 고친 뒤 영구 저장된 폴더 핸들을 재사용할 수 있는지가 접근 권한에 따라 달라집니다. 대체 방식에서는 선택한 파일이 세션 동안만 유지되므로 새로 고칠 때마다 반드시 다시 선택해야 합니다.

## 대규모 라이브러리

카탈로그는 데이터를 디스크에 저장하고 작업을 페이지나 크기가 제한된 배치로 나누므로 큰 컬렉션 전체를 한 번에 메모리에 올릴 필요가 없습니다. 규모 측정과 고정 기준 측정은 선택적으로 실행하는 로컬 개발 진단입니다. commit, release, `verify`, GitHub Actions의 통과 조건이 아니며 일반적인 성능 보장도 아닙니다. 스캔 시간과 실제 한계는 저장 장치 속도, 사용 가능한 메모리, 메타데이터, 아트워크, 브라우저 또는 운영 체제의 제한에 따라 달라집니다.

트랙 목록을 스크롤하는 동안 EffeTune은 현재 위치와 가까운 페이지를 캐시에 보관합니다. 모바일 레이아웃에서는 현재 스크롤 방향으로 최대 두 페이지를 미리 읽고, 추가 미리 읽기보다 화면에 필요한 페이지를 우선하며, 표시 범위가 겹치는 트랙 행을 재사용합니다. 스크롤이 계속되는 동안에도 화면 표시를 위해 완료된 읽기는 이 제한된 캐시에 즉시 반영됩니다. 표시 위치 요청은 가장 최근 위치로 합쳐지며, 방금 읽은 페이지에 그 위치가 포함되어 있으면 데이터베이스를 다시 읽지 않습니다. 더 이상 필요하지 않은 대기 중 미리 읽기는 버립니다. SQLite 자체는 중단을 지원하지만, 현재 카탈로그 어댑터는 각 명령문을 동기식으로 실행하며 다른 worker에서 중단할 수 있는 경로를 제공하지 않습니다. 따라서 특히 저장 장치가 느릴 때 매우 빠르게 먼 위치로 이동하면 진행 중인 읽기가 끝날 때까지 잠시 빈 영역이 보일 수 있습니다.

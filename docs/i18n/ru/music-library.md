# Как пользоваться музыкальной библиотекой

В версии 2.1.0 представлена музыкальная библиотека с поддержкой CUE, использующая схему каталога v3. Папки и плейлисты из прежней библиотеки в этот каталог не переносятся: добавьте папки заново, а плейлисты создайте или импортируйте повторно. Прежний каталог и аудиофайлы не изменяются.

**Музыкальная библиотека** индексирует выбранные музыкальные папки и позволяет искать локальную коллекцию по трекам, альбомам, исполнителям, жанрам, подпапкам, папкам, недавно добавленным трекам и плейлистам. Воспроизведение из библиотеки, как и обычное воспроизведение музыкальных файлов, проходит через текущий конвейер эффектов EffeTune.

**Музыкальная библиотека** хранит внутренний каталог приложения, кэш обложек и плейлисты. Она не редактирует, не переименовывает, не перемещает и не удаляет сами музыкальные файлы.

## Доступные среды

- **Настольное приложение:** использует полнофункциональный сканер папок, а выбранные папки остаются доступными при следующем запуске. В настольной версии также можно показывать треки в папке, где находится файл.
- **Браузеры Chromium на ПК с File System Access:** постоянно сохраняют дескриптор выбранной папки. После предоставления доступа его можно использовать снова после перезагрузки, хотя браузер может повторно запросить разрешение.
- **Мобильные браузеры, Safari, Firefox и другие браузеры без File System Access:** сохраняют выбранные объекты `File` только в текущем сеансе страницы. Каталог остаётся сохранённым, но после перезагрузки файлы нельзя открыть снова. После каждой перезагрузки заново выберите папку или файлы; EffeTune привяжет их к существующим записям по нормализованному относительному пути.

**Музыкальная библиотека** индексирует распространенные расширения медиафайлов, включая MP3, WAV, OGG, FLAC, Opus, M4A, AAC, WebM и MP4. С помощью внешнего файла CUE она также может разделить на треки альбомный WAV- или FLAC-файл из той же папки. В файлах MP4 EffeTune воспроизводит только звуковую дорожку и не показывает видео. Возможность воспроизведения, включая поддержку аудиокодека внутри MP4, также зависит от средств декодирования браузера и ОС.

## Как открыть музыкальную библиотеку

- **Интерфейс для ПК:** нажмите кнопку **Музыкальная библиотека** в заголовке.
- **Мобильный интерфейс:** откройте вкладку **Библиотека** в нижней навигации.
- **Настольное приложение:** также можно открыть через **Вид > Музыкальная библиотека** или **Ctrl+L** (на macOS - **Command+L**).

Чтобы вернуться к редактированию эффектов, в интерфейсе для ПК нажмите кнопку **Effect Pipeline**, а в мобильном интерфейсе вернитесь на вкладку **Эффекты**. В настольном приложении также можно использовать **Вид > Effect Pipeline** или **Ctrl+E** (на macOS - **Command+E**).

Если вы хотите, чтобы при запуске сначала открывалась **Музыкальная библиотека**, откройте **Настройки > Конфигурация...** и в поле **Вид при запуске:** выберите **Музыкальная библиотека**. В соседнем списке выберите, какой раздел библиотеки открывать первым: **Треки**, **Альбомы**, **Исполнители**, **Жанры** или **Подпапки**.

## Как добавить музыкальную папку

1. Откройте раздел **Музыкальная библиотека**.
2. Выберите **Добавить папку с музыкой**.
3. Выберите папку с музыкальными файлами. В мобильных браузерах и браузерах с резервным режимом вместо постоянного доступа к папке может открыться экран выбора файлов внутри папки.
4. Дождитесь завершения сканирования. В строке состояния отображаются количество треков и альбомов, а во время индексирования - ход выполнения.

Если вы попытаетесь добавить папку, которая уже находится внутри зарегистрированной папки, EffeTune предупредит об этом и не будет индексировать ее повторно. Если добавить родительскую папку, содержащую уже зарегистрированные папки, существующие папки можно объединить с новой.

## Просмотр и поиск

Переключайте каталог вкладками навигации.

- **Треки** - показывает все проиндексированные треки. В интерфейсе для ПК это сортируемая таблица, в мобильном интерфейсе - компактный список.
- **Альбомы** - группирует треки по альбомам из метаданных.
- **Исполнители** - группирует треки по исполнителям или исполнителям альбомов из метаданных.
- **Жанры** - группирует треки по жанрам из метаданных.
- **Подпапки** - группирует треки по относительному пути подпапки, непосредственно содержащей файл, внутри каждой проиндексированной музыкальной папки.
- **Папки** - показывает зарегистрированные корневые музыкальные папки и состояние сканирования.
- **Недавно добавлено** - показывает недавно проиндексированные треки.
- **Плейлисты** - показывает плейлисты, созданные или импортированные в разделе **Музыкальная библиотека**.

Значение исполнителя альбома, разделённое точками с запятой, например `Artist A; Artist B`, индексируется для каждого исполнителя, а полная подпись сохраняется на экране. `&`, `/` и `feat.` не считаются разделителями.

Например, файл `Artist/Album/01 Song.flac` отображается в группе подпапки `Artist/Album`. Одинаковые относительные пути в разных проиндексированных корнях остаются отдельными. Файлы непосредственно в корне не создают группу подпапки, но по-прежнему доступны в разделах **Треки** и **Папки** для этого корня.

Поле **Поиск в библиотеке** позволяет искать сразу по трекам, альбомам, исполнителям и плейлистам. В интерфейсе для ПК список треков можно сортировать из заголовка по **Название**, **Исполнитель**, **Альбом**, **Жанр** и **Время**. В разделах альбомов, исполнителей, жанров, подпапок и плейлистов доступен список **Сортировка**, работающий с каталогом. В зависимости от раздела можно выбрать сортировку по названию, исполнителю, пути, количеству треков, общей длительности, времени обновления или времени создания — по возрастанию или убыванию. Каждый раздел сохраняет собственный выбор.

При поиске треков запросы из трёх и более символов могут совпадать с любой частью названия, имени исполнителя, альбома, жанра, имени файла или пути. Запросы из одного или двух символов совпадают только с началом слова. Для поиска внутри слова введите не менее трёх символов.

И в интерфейсе для ПК, и в мобильном интерфейсе, если поиск треков или страница альбома, исполнителя, жанра, подпапки либо плейлиста содержит не более 300 треков, все они выбираются по умолчанию. При 301 треке и более автоматического выбора нет. Используйте флажки строк, **Выбрать всё** или **Снять выделение**, чтобы изменить выбор.

На мобильном устройстве сначала показывается обычный список названий без столбцов исполнителя и длительности. Только удерживание трека включает режим выбора; появятся флажки, **Выбрать всё** и **Снять выделение**, а обычные действия со строками останутся доступны. Автоматический выбор и последующие изменения, включая **Выбрать всё**, **Снять выделение** и отдельные флажки, меняют только состояние выбора и не включают или не выключают режим выбора.

Если метаданные отсутствуют или не читаются, EffeTune использует для отображения имя файла и сведения о папке. В свойствах трека можно проверить путь к файлу, формат, частоту дискретизации, битовую глубину, битрейт и основные поля метаданных. Для трека CUE также показаны его тип, путь к файлу CUE, путь к исходному аудиофайлу и диапазон трека внутри него.

## Альбомные файлы с CUE

Поместите внешний файл `.cue` рядом с указанными в нем файлами WAV или FLAC, затем добавьте или повторно просканируйте эту папку. Каждая допустимая запись `TRACK ... AUDIO` появится в Музыкальной библиотеке как отдельный трек. Если они указаны, EffeTune использует название, исполнителя, дату, жанр и номер трека из CUE, а технические сведения получает из исходного WAV- или FLAC-файла.

Для композиций, добавленных в Музыкальную библиотеку, EffeTune сначала использует обложку, встроенную в исходный аудиофайл. Если её нет, программа ищет рядом с файлом CUE, по порядку, `cover.jpg`, `cover.png`, `front.jpg`, `front.png`, а затем JPEG- или PNG-файл с именем исходного аудиофайла — с его звуковым расширением или без него. При прямом воспроизведении в настольном приложении автоматически используются те же соседние файлы изображений; в этом режиме встроенная обложка из исходного аудиофайла не извлекается. При прямом воспроизведении в браузере используется подходящее изображение, доступное через выбранные файлы или зарегистрированную папку.

Альбом CUE также можно воспроизвести напрямую с помощью **Open music files**, а на мобильном устройстве — **Open Music**. В настольном приложении доступна и команда **File > Open music file...**; выберите только файл `.cue`. В Chromium-браузере на ПК сначала добавьте папку альбома в Музыкальную библиотеку и разрешите доступ. После этого достаточно выбрать только файл `.cue`: EffeTune откроет указанные в нём WAV- или FLAC-файлы и подходящую обложку из зарегистрированной папки, не добавляя этот выбор в каталог. В браузерах без File System Access по-прежнему нужно передать файл `.cue` вместе со всеми и только указанными в нём WAV- или FLAC-файлами и, при необходимости, подходящей обложкой. Допустимый выбор заменяет текущую очередь. Если проверка не пройдена, очередь останется без изменений.

Если файл CUE недопустим или не позволяет надежно определить исходные файлы, EffeTune объясняет проблему и импортирует WAV или FLAC как обычные треки на всю длину файла. Исправьте файл CUE или имена файлов и просканируйте папку снова.

## Воспроизведение из библиотеки

Выберите трек, альбом, исполнителя, жанр, подпапку, папку, результат поиска или плейлист, затем используйте следующие действия.

- **Воспроизвести** - заменяет текущую очередь плеера и начинает воспроизведение.
- **Перемешать** - воспроизводит выбранные треки в случайном порядке.
- **Воспроизвести следующим** - вставляет выбранные треки после текущего трека.
- **Добавить в очередь** - добавляет выбранные треки в конец очереди.
- **Добавить в плейлист** - сохраняет выбранные треки в плейлист раздела **Музыкальная библиотека**.

На ПК можно дважды щелкнуть строку трека, чтобы начать воспроизведение с этой позиции, а действия с треком открыть правым щелчком или через меню **Еще**. На мобильном устройстве коснитесь трека в обычном списке, чтобы воспроизвести его; удерживание включает описанный выше режим выбора.

Обычные элементы управления музыкальным плеером и настройки повтора/случайного воспроизведения продолжают работать. На устройствах с клавиатурой также доступны обычные сочетания клавиш плеера. Если папка стала недоступна и трек из библиотеки нельзя открыть, подключите эту папку заново или импортируйте ее повторно.

## Обновление и повторное подключение папок

После добавления, удаления, переименования файлов или изменения тегов в музыкальной папке используйте **Пересканировать**. Повторное сканирование обновляет измененные треки, убирает из каталога файлы, которые больше не найдены, и также пытается снова сопоставить ранее недоступные элементы плейлистов.

В представлении **Папки** по состоянию видно, доступна ли папка.

- **OK** - папка доступна.
- **Не сканировано** - папка еще не индексировалась.
- **Не найдено** - папка или сохраненный путь недоступны.
- **Подключить заново** - EffeTune снова требуется разрешение на доступ.

Если у папки показано **Подключить заново**, выберите **Подключить заново** и снова разрешите доступ к той же папке. Удаление папки только убирает ее из каталога раздела **Музыкальная библиотека**; файлы на диске не удаляются.

## Плейлисты

Плейлисты раздела **Музыкальная библиотека** сохраняются внутри EffeTune и могут содержать треки из проиндексированных папок.

Доступны следующие действия.

- Создать плейлист из выбранных треков библиотеки.
- Сохранить текущую очередь плеера как плейлист.
- Переименовывать, дублировать, удалять и менять порядок плейлистов.
- Перетаскивать треки внутри плейлиста, чтобы изменить порядок. В средах, где перетаскивание неудобно, используйте **Переместить вверх** и **Переместить вниз**.
- Использовать **Импортировать плейлист** для плейлистов в форматах M3U, M3U8, PLS и XSPF.
- Открыть конкретный плейлист и экспортировать его через **Экспортировать M3U8** или **Экспортировать XSPF**.

### Недавно воспроизведённые и Избранное

EffeTune показывает два специальных плейлиста рядом с обычными в одной сетке карточек. Они создаются только при необходимости: **Недавно воспроизведённые** — при запуске проиндексированного трека, а **Избранное** — когда вы впервые отмечаете трек звездой.

- **Недавно воспроизведённые** хранит 100 последних неповторяющихся треков, начиная с самого нового. Повторное воспроизведение возвращает трек в начало списка.
- **Избранное** содержит треки, отмеченные значком ☆. На ПК используйте звезду рядом с треком, а на мобильном устройстве откройте меню **Ещё** для этого трека. На ПК то же меню можно открыть правой кнопкой мыши.

Названия этих плейлистов фиксированы и отображаются на текущем языке интерфейса, поэтому переименовать их нельзя. Их можно дублировать, экспортировать и удалять, как обычные плейлисты. После удаления плейлист создаётся заново пустым, когда он снова потребуется для истории воспроизведения или избранного. В области обложки на карточках отображается значок часов или звезды; кнопка воспроизведения в правом нижнем углу карточки **Избранное** сразу запускает плейлист. Специальные плейлисты не входят в результаты поиска по обычным плейлистам.

При сканировании папки EffeTune автоматически импортирует поддерживаемые файлы плейлистов после индексации композиций и пропускает файлы, содержимое которых не изменилось. Если содержимое файла в той же папке и по тому же относительному пути изменилось, EffeTune атомарно заменяет элементы автоматически импортированного плейлиста; при этом заменяются и изменения элементов, сделанные в EffeTune. Неудачный или отменённый импорт повторяется при следующем сканировании. Удаление или переименование исходного файла не удаляет существующий плейлист, а переименованный файл импортируется как новый плейлист.

При импорте показывается предварительный просмотр с количеством элементов, сопоставленных с треками в текущей библиотеке. Элементы без совпадений по возможности сохраняются как несопоставленные, чтобы их можно было сопоставить позже, когда соответствующая папка будет добавлена или подключена заново.

Если при экспорте выбрать **Относительные пути**, пути по возможности записываются относительно места экспорта. Это удобно, если вы хотите переносить плейлист вместе с музыкальной папкой. Форматы M3U8 и XSPF не сохраняют диапазон трека CUE внутри файла альбома, поэтому EffeTune исключает такие треки и сообщает их количество. Вместо исключенного трека физический путь к файлу альбома не записывается.

## Безопасность и место хранения

- **Музыкальная библиотека** читает музыкальные файлы и метаданные, но не записывает изменения в музыкальные файлы.
- Кэш обложек и плейлисты являются данными приложения, а не изменениями, встроенными в музыкальные файлы.
- Группы подпапок создаются из относительных путей, сохранённых в каталоге.
- Хранилище браузера может быть очищено настройками браузера или действиями пользователя. При необходимости экспортируйте важные плейлисты.
- В браузерах с File System Access разрешения определяют, можно ли повторно использовать сохранённый дескриптор папки после перезагрузки. В резервном режиме выбранные файлы действуют только в текущем сеансе, поэтому после перезагрузки их всегда нужно выбирать снова.

## Большие библиотеки

Каталог хранит данные на диске и обрабатывает их страницами или ограниченными пакетами, поэтому большую коллекцию не требуется целиком загружать в память. Масштабные измерения и измерения на фиксированной эталонной системе — необязательная локальная диагностика для разработки. Они не являются условием для commit, release, `verify` или GitHub Actions и не гарантируют производительность в общем случае. Время сканирования и практические ограничения зависят от скорости накопителя, доступной памяти, метаданных, обложек и ограничений браузера или операционной системы.

При прокрутке списка треков EffeTune сохраняет ближайшие страницы в кэше. В мобильном интерфейсе приложение заранее читает до двух страниц по направлению прокрутки, отдаёт приоритет странице, которая нужна на экране, перед дополнительным упреждающим чтением и повторно использует перекрывающиеся видимые строки. Даже если прокрутка продолжается, завершённые чтения для области просмотра сразу публикуются в этом ограниченном кэше. Запросы позиции объединяются до самого нового; если он попадает на только что загруженную страницу, дополнительное чтение базы данных не выполняется. Ожидающие упреждающие чтения, которые больше не нужны, отбрасываются. SQLite поддерживает прерывание, но текущие адаптеры каталога выполняют каждый оператор синхронно и не предоставляют способ прервать его из другого worker. Поэтому при исключительно быстром переходе всё ещё может ненадолго появиться пустой участок до завершения текущего чтения, особенно на медленном накопителе.

# EffeTune FAQ

EffeTuneはオーディオ愛好家向けのリアルタイムDSP（デジタル・シグナル・プロセッサ）アプリです。ウェブ版（ブラウザ）とデスクトップ版（Electron）の2系統を提供しており、どちらも同一のエフェクト群・操作体系で動作します。本ドキュメントでは導入・設定・トラブルシューティング・マルチチャンネル運用・エフェクト操作・周波数特性補正までを網羅的に解説します。

## 1. ストリーミング再生のための初期セットアップ

Windows環境でSpotify→VB-CABLE→EffeTune→DAC/AMPによる再生を例に説明します。他のストリーミングサービスやOSでも概念は同じです。

### 1.1. VB-CABLEの導入と96 kHz化
VB-CABLE Driver Pack45をダウンロードしてVBCABLE_Setup_x64.exeを管理者権限で実行し、PCを再起動します。  
再起動直後、OSの既定出力がCABLE Inputに変更されていますので、設定→システム→サウンド→出力で元のスピーカー／DACを既定に戻します。  
設定→システム→サウンドを開き、出力のCABLE Input/入力のCABLE Outputの形式をともに24-bit, 96 000 Hzに設定します。  
VBCABLE_ControlPanel.exe（VB Cableのインストールフォルダやzipに含まれる）を管理者権限で起動し、Menu▸Internal Sample Rate=96000 Hzを選択後Restart Audio Engineをクリックします。  
管理者権限で実施しないと再起動時に48 kHzに戻ります。必ず管理者権限で変更してください。  
VB Cable使用時、24 kHz以上に折り返しノイズが発生する場合は48 kHz動作が原因です。必ず96 kHz化を確認してください。

### 1.2. ストリーミングサービス入力(Spotify例)
設定→システム→サウンド→音量ミキサーを開き、アプリ一覧からSpotify.exeの出力デバイスをCABLE Inputに変更します。
この時点で曲を再生し、無音になっていればルーティング成功です。
macOSでは Rogue Amoeba製の **SoundSource** などを使い、Spotifyの出力を同様に **CABLE Input** に割り当ててください.

### 1.3. EffeTuneのオーディオ設定
EffeTuneを開き、**設定** メニューから **オーディオ設定** を開きます。
* 入力デバイス:CABLE Output(VB-Audio Virtual Cable)  
* 出力デバイス:物理DAC/スピーカー  
* Sample Rate=96 000 Hz(96 kHz未満では内部処理が最適化されず、特に歪み系エフェクトで音質が劣化)

### 1.4. 動作確認
Spotifyを再生したままEffeTuneのマスターON/OFFを切り替え、音質変化を確認します。

## 2. トラブルシューティング

### 2.1. オーディオ再生品質の問題

| 症状 | 対処 |
| ------ | ------ |
| ドロップアウト／グリッチ | 1)**設定** メニューまたはモバイルのオーバーフローメニューから **オーディオをリセット** を選び、オーディオパイプラインを再構築。デスクトップ版では表示▸リロードも使用可。2)アクティブなエフェクト数を減らす。3)オーディオ設定▸レイテンシを大きめに設定し安定性を向上(映像との同期ズレ注意)。 |
| 音が歪む／クリップ | チェーン末尾にLevel Meterを挿入し0 dBFSを超えないよう調整。必要に応じBrickwall LimiterをLevel Meter直前に追加。 |
| 20 kHz以上に折り返しノイズ | VB-CABLEの内部サンプルレートが48 kHzのまま。初期セットアップを再確認。 |

### 2.2. CPU使用率が高すぎる
使用していないエフェクトを無効化するかEffect Pipelineから外してください。

### 2.3. エコーが発生する
入出力デバイスが誤ってループバック構成になっている可能性。EffeTune出力が再びEffeTune入力へ戻っていないか確認。

### 2.4. 入力／出力／エフェクトの不具合

| 症状 | 対処 |
| ------ | ------ |
| オーディオ入力がない | 1)再生アプリが正しくCABLE Inputへ出力しているか確認。2)ブラウザではマイク権限を許可し、**オーディオ設定** で入力デバイスにCABLE Outputを選択。 |
| エフェクトが動作しない | 1)マスター/各エフェクト/SectionのON/OFF確認。2)パラメータを初期値にリセットし再調整。 |
| オーディオ出力がない | **オーディオ設定** を確認。ブラウザが出力デバイス選択に対応していない場合は、OSまたはブラウザの既定出力を物理DAC/AMPに設定。 |
| 他プレーヤーでCABLE Input in useエラー | 他アプリがCABLE Inputを占有していないか確認。 |

### 2.5. マルチチャンネル出力の不整合
EffeTuneは1→2→…→8 ch順で信号を出力しますが、Windowsスピーカー構成が4 chの場合、リア(3+4 ch)がセンター+サブに割り当てられることがあります。  
**回避策:** オーディオデバイス構成を7.1 chに設定し、EffeTune出力も8 chに設定、リア信号を5,6 chとして扱う。

## 3. マルチチャンネル&ハードウェア接続

### 3.1. HDMI+AVアンプ
PCのHDMI出力を7.1 chとして設定しAVアンプのHDMI入力へ接続。EffeTuneからは最大8 chまで送出可能。  
**メリット:** ケーブル1本で8 ch伝送できコスパ高。既存機器との切替え容易。  
**デメリット:** 古いAVアンプで音質劣化例あり。チャンネルリマッピング機能が意図せず働く場合あり。

### 3.2. マルチチャンネル非対応インターフェイス(MOTU M4等)
Out 1-2/Out 3-4を別デバイスとして認識し4 ch一括出力不可。  
**対応方法:**  
* Voicemeeterでマルチチャンネル出力を受けASIOドライバー経由でM4の4 chを統合  
* ASIO Link Proで4 chを1仮想デバイスとして公開(高難度だが柔軟)

### 3.3. チャンネルディレイ&タイムアライメント
Basic▸MultiChannel PanelまたはDelay▸Time Alignmentで各チャンネルを10 µs単位(最小1サンプル)で遅延。Bluetoothリア等で遅延が大きい場合はフロント側を100-400 ms遅延。マイナス遅延不可のため映像同期はプレーヤー側で映像を遅延。

### 3.4. 8ch上限と拡張要望
現行Windows/macOS/Linuxの標準ドライバは8 ch(7.1)出力が上限。10 ch以上はOS対応次第、EffeTune側で対応容易。

## 4. よくある質問(FAQ)

| 質問 | 回答 |
|------|------|
| PWA版はどの端末で使える? | Androidスマートフォン/タブレット、iPhone/iPad、Windows、macOS、Linux、ChromeOSなど、主要なモバイル/デスクトップ環境で利用できます。PWAのため端末固有アプリではなくブラウザ上で動作しますが、インストール方法、オーディオ入出力デバイス選択、対応音楽形式はブラウザとOSに依存します。 |
| PWA版をインストールできない | EffeTuneサイトの **PWA版をインストール** ボタン、またはWeb版右上の歯車メニューから **アプリをインストール** を選びます。うまく表示されない場合、AndroidやPCではChromeまたはEdgeなどのChrome系ブラウザで開いてください。iPhone/iPadではSafariで開き、共有メニューからホーム画面に追加してください。アプリ内ブラウザ、プライベートブラウズ、古いブラウザではインストール項目が出ない場合があります。 |
| サラウンド入力(5.1 ch等)対応? | Web Audio API制限で入力は最大2 ch。出力/エフェクトは8 ch対応。 |
| 推奨エフェクトチェーン長? | CPUが許す限り自由。音切れ・レイテンシーが出ない範囲で無制限。 |
| 最高音質を得るには? | 96 kHz以上を使用し控えめ設定から開始。チェーン末尾にLevel Meterを挿入しヘッドルーム監視、必要に応じBrickwall Limiterで歪み防止。 |
| どんなソースでも動作? | はい。仮想オーディオデバイス経由でストリーミング/ローカル/物理機器等任意ソース処理可能。 |
| DRM（著作権保護）付きコンテンツを再生できますか？ | 直接再生することはできません。EffeTuneは音声を加工するアプリですが、著作権保護付きコンテンツは提供元が認めた環境での再生を前提としており、第三者の音声加工アプリでは利用できない場合があります。提供元の公式アプリまたはウェブサイトを、利用規約と対応する音声出力方法に従ってお使いください。EffeTuneが著作権保護を解除または回避することはありません。 |
| 音声入力なしで音楽ファイルプレーヤーだけ使える? | はい。起動後にマイク音声がイヤホン等へ回り込む場合は、**オーディオ設定** の **入力デバイス** で **なし（音楽ファイルプレーヤー専用）** を選んでください。EffeTuneは無音ソースでパイプラインを維持するため、プレーヤーや **Oscillator** などの信号生成エフェクトも動作します。音声入力を選ぶと、USBオーディオインターフェイス等を接続して外部機器の音を処理したり、入力信号を **Spectrum Analyzer** などで確認したりできます。 |
| モバイルWebアプリで他アプリの音声を処理できる? | 通常はできません。モバイルブラウザは他アプリ音声の汎用ループバック入力を提供しないため、モバイルではEffeTune内蔵プレイヤー中心の利用になります。 |
| 対応している音楽ファイル形式は? | ブラウザとOSの音声デコード機能に依存します。目安としてMP3、WAV、AAC/M4Aは多くの環境で使いやすく、FLAC、OGG/Vorbis、Opus/WebMなどは環境により対応状況が異なります。MP4は映像を表示せず音声トラックのみを再生します。再生可否はMP4内の音声コーデックにも依存し、AACが一般的な目安です。再生できない場合はMP3、AAC/M4A、WAVを試してください。 |
| 複数の音楽ファイルを再生できる? | はい。**音楽ファイルを開く** で表示される端末標準のファイル選択画面で複数ファイルを選んでから開くと、プレイリストとして読み込まれます。複数選択やフォルダ内の全ファイル指定が使えるかは、端末、ブラウザ、ファイル選択画面に依存します。 |
| Music Libraryでは何ができる? | 選択した音楽フォルダをインデックス化し、曲、アルバム、アーティスト、ジャンル、曲が直接入っているサブフォルダで閲覧・検索してEffeTuneで再生できます。保存されるのはアプリ内のライブラリ情報やプレイリストで、音楽ファイル自体ではありません。 |
| Music Libraryはどの環境で使える? | デスクトップ版ではフォルダスキャンをそのまま利用できます。Chromium系ブラウザでは対応環境でFile System Accessを使います。Safari/Firefoxではインポート方式のフォールバックになるため、リロード後や権限が切れた後はフォルダまたはファイルの選択が再度必要になる場合があります。 |
| ライブラリのフォルダを更新・再接続するには? | ファイルを追加、削除、編集した後は **Music Libraryを再スキャン** を使います。フォルダがアクセス不可として表示されたら、そのフォルダの **再接続** ボタンを押し、同じフォルダへのアクセスを許可し直します。 |
| Music Libraryで読み書きできるプレイリスト形式は? | M3U、M3U8、PLS、XSPF形式のプレイリストをインポートでき、M3U8またはXSPF形式でエクスポートできます。 |
| Music Libraryはディスク上の音楽ファイルを書き換える? | いいえ。スキャン、メタデータ読み取り、アートワークキャッシュ、プレイリスト編集、再生操作はアプリ内で完結し、ディスク上の音楽ファイルを変更することはありません。 |
| Webアプリで出力デバイスを選べない | ブラウザ対応と権限に依存します。Chrome/Chromiumの安全なページで試すか、使いたいDAC/AMPをOSまたはブラウザの既定出力にしてください。 |
| Sample RateやOutput Channelsが指定値にならない | ブラウザやデバイスが非対応値を丸めたり無視したりする場合があります。EffeTuneは実際に有効になった値で動作します。 |
| Webプレイヤーはプレイリストを記憶する? | リピート/シャッフル設定は保存されますが、通常のファイル選択はブラウザ制約によりリロード後に復元されません。 |
| 画面オフ中もモバイル再生できる? | ブラウザ依存で、特にiOSでは安定保証できません。対応環境ではWake Lockを使いますが、バックグラウンド再生は保証されません。 |
| EffeTuneの省電力モードはどう違う? | Web/PWA版とElectronデスクトップ版の両方で利用できます。**設定** → **省電力** で選びます。**バックグラウンド処理優先** は外部入力が無音でも音声処理を継続します。**標準省電力（既定）** は選択中の音声入力を原則保持しながら、無音時のDSP処理と画面更新を抑えます。**最大省電力** は、未使用またはバックグラウンドで無音の音声入力を設定時間後に停止することもできます。現在の経路で安全性を確認できる場合は、再生位置を進めたままDSPをバイパスしたり、出力をゼロに保ったりすることがあります。個別の状態表示はなく、ユーザー操作が必要な場合にだけメニューへ **音声処理を再開** または **音声入力を再開** が表示されます。 |
| 「無音判定レベル」と「音声入力を停止するまで」は何を変える? | **無音判定レベル**（-90～-20 dBFS、10 dB刻み）は、入力と出力の測定電力を無音とみなす境界です。値を低くするほど、小さな音を無音と判定しにくくなります。**最大省電力** の **音声入力を停止するまで**（1分/5分/15分/**停止しない**）は、マイクなどの入力解放だけを制御します。経路のない音声グラフを短時間でSuspendedにする処理とは独立しているため、入力を保持したままグラフだけがSuspendedになる場合があります。 |
| Web/PWA版を非表示にしても「バックグラウンド処理優先」なら必ず処理を続ける? | いいえ。EffeTune側では外部入力経路の無音による自動Suspendedを避け、処理継続を優先しますが、非表示ページをfreeze、suspend、破棄するかはブラウザとOSが最終的に決めます。**最大省電力** で入力を停止した場合、ページへ戻っただけ、または外部信号が戻っただけでマイク権限を自動要求しません。明示的なユーザー操作で **音声処理を再開** を選んでください。 |
| AVアンプとインターフェイスどちらが経済的? | 既機器活用ならHDMI接続AVアンプが手軽。PC中心ならマルチチャンネル対応インターフェイス+小型D級アンプがコスト・音質両面有利。 |
| VB-CABLE導入直後に他アプリの音が出ない | OS既定出力がCABLE Inputに切替ったため。サウンド設定で元へ戻す。 |
| 4 ch分離後に3+4 chのみボリューム制御 | VolumeエフェクトをSplitter下流に配置しChannelを3+4に設定。分岐前に置くと全chが変化。 |
| 共有されたIR Reverbがdry音だけになる、またはIRが見つからないと表示される | URLやプリセットにはIRのコンテンツIDだけが保存され、音声ファイルは含まれません。元と同じファイルを取り込むと自動再リンクできます。別のIRを使う場合は **Impulse Response Library** から置き換えてください。IRがない状態ではwet音は出ないため、**Dry** の設定も確認してください。 |
| 別の人にIR Reverbの結果を再現してもらうには? | プリセットまたはURLと一緒に、元のIRファイルと出典・ライセンス情報を渡してください。受け取った側が未変更の同じファイルを取り込むと同じIDになります。サンプルレート、出力チャンネル選択、IR Reverbの各設定も揃えてください。 |

## 5. 周波数特性補正・音場補正

### 5.1. 15Band PEQへAutoEQ設定インポート
EffeTune v1.51以降、右上ボタンからAutoEQイコライザ設定を直接インポート可能。

### 5.2. 測定・補正設定のPipelineペースト
測定画面でコピーした5Band PEQ設定をEffect PipelineビューでCtrl+Vペースト。メニュー貼付でも同様。

## 6. エフェクト操作Tips
* 信号フローは最上段→最下段。  
* Matrixエフェクトで2→4 chや8→2 ch変換を自由構築(バスルーティングでChannel=All)。  
* MultiChannel Panelで8 ch分のレベル/ミュート/ディレイを一括管理。
* マルチチャンネルのリバーブセンド/リターンを作る場合は、**Matrix**で必要なソースチャンネルを空きバスへコピーし、そのバスの**IR Reverb**を**Dry** -96 dB、**Wet** 0 dBにします。Matrixのゲインをセンド量として使い、wetバスは必要な出力チャンネルだけへ戻します。元のdry経路を二重にしないことがポイントです。

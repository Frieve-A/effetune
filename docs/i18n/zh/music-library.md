# 音乐库使用方法

版本2.1.0引入了支持CUE、采用目录架构v3的音乐库。旧音乐库中登记的文件夹和播放列表不会转入此目录，请重新添加音乐文件夹，并重新创建或导入播放列表。旧目录和音频文件都不会被修改。

音乐库会为您选择的音乐文件夹建立索引，让您可以按曲目、专辑、艺人、流派、子文件夹、文件夹、最近添加的曲目和播放列表浏览本地音乐收藏。播放的声音和普通音乐文件播放一样，会经过当前 EffeTune 的 Effect Pipeline。

音乐库保存的是应用内的目录、封面缓存和播放列表。它不会编辑、重命名、移动或删除音乐文件本身。

## 可用环境

- **桌面应用：** 使用完整功能的文件夹扫描器，所选文件夹在下次启动时仍可使用。桌面版还可以在文件所在文件夹中显示曲目。
- **支持 File System Access 的 PC Chromium 浏览器：** 持久保存所选文件夹的句柄。授予访问权限后，重新加载页面仍可复用该句柄，但浏览器可能再次请求授权。
- **移动浏览器、Safari、Firefox 及其他不支持 File System Access 的浏览器：** 仅在当前页面会话中保留所选的 `File` 对象。目录会继续保存，但重新加载后无法再次打开这些文件。每次重新加载后都要重新选择文件夹或文件；EffeTune 会通过规范化的相对路径，将它们重新连接到现有目录项。

音乐库会索引MP3、WAV、OGG、FLAC、Opus、M4A、AAC、WebM、MP4等常见媒体文件扩展名。它还可以通过外部CUE表，将同一文件夹中的WAV或FLAC专辑文件拆分成独立曲目。对于MP4文件，EffeTune只播放音轨，不显示视频。实际能否播放（包括是否支持MP4内的音频编解码器）仍取决于浏览器和操作系统的解码能力。

## 打开音乐库

- **PC 布局：** 点击页眉中的 **音乐库** 按钮。
- **移动端布局：** 打开屏幕底部导航中的 **音乐库** 标签。
- **桌面应用：** 也可以通过 **视图 > 音乐库** 或 **Ctrl+L**（macOS 为 **Command+L**）打开。

要返回效果编辑，PC 布局中点击 **Effect Pipeline** 按钮，移动端布局中返回 **效果** 标签。桌面应用中也可以使用 **视图 > Effect Pipeline** 或 **Ctrl+E**（macOS 为 **Command+E**）。

如果想在启动时首先显示音乐库，请打开 **设置 > 配置...**，将 **启动时显示:** 设置为 **音乐库**。在 **音乐库** 旁边的下拉列表中，可选择首先显示的项目：**曲目**、**专辑**、**艺人**、**流派** 或 **子文件夹**。

## 添加音乐文件夹

1. 打开音乐库。
2. 选择 **添加音乐文件夹**。
3. 选择包含音乐文件的文件夹。在移动端或使用回退方式的浏览器中，可能会出现选择文件夹内文件的界面，而不是持久文件夹访问。
4. 等待扫描完成。状态行会显示曲目数和专辑数，建立索引期间也会显示进度。

如果尝试添加已注册文件夹内部的文件夹，EffeTune 会发出警告，并且不会重复建立索引。添加包含已注册文件夹的父文件夹时，可以将现有文件夹合并到新的文件夹中。

## 浏览和搜索

使用导航标签切换分类视图。

- **曲目** - 显示所有已索引的曲目。PC 布局中是可排序表格，移动端布局中是紧凑列表。
- **专辑** - 根据元数据按专辑汇总。
- **艺人** - 根据元数据中的艺人或专辑艺人汇总。
- **流派** - 根据元数据中的流派汇总。
- **子文件夹** - 按每个曲目文件直接所在的子文件夹相对于各个已索引音乐文件夹根目录的路径分组。
- **文件夹** - 显示已注册的音乐文件夹根目录、扫描状态和实际目录树。
- **最近添加** - 显示最近索引的曲目。
- **播放列表** - 显示在音乐库内创建或导入的播放列表。

对于像 `Artist A; Artist B` 这样以分号分隔的专辑艺人值，系统会分别归入每位艺人，同时保留完整署名用于显示。`&`、`/` 和 `feat.` 不会被视为分隔符。

在 **文件夹** 中打开一个根目录，即可使用 **树状视图** 浏览。子文件夹显示在曲目列表上方；选择子文件夹可进入其中，使用面包屑路径或 **上一页** 可返回上一级。曲目列表只显示直接存放在当前层级的曲目。切换到 **平铺视图** 后，则会显示该已注册根目录中的所有已索引曲目，包括所有子文件夹中的曲目。切换视图时会保留当前的面包屑位置，因此返回 **树状视图** 后可从原位置继续浏览。**搜索音乐库** 始终搜索整个音乐库，所以搜索期间不会显示子文件夹列表。

**文件夹** 树严格按实际相对路径区分目录，并包含扫描发现的每一条目录曲目记录，即使其元数据仍在分析中或分析失败。文件夹旁的数量包括该文件夹及其所有下级目录中的曲目记录。CUE 曲目按逻辑曲目分别计数；多个逻辑曲目即使共用一个 WAV 或 FLAC 源文件，也会计为多首曲目，而不是一个物理音频文件。**子文件夹** 的含义不同：它按规范化后的直接父目录标识，对已成功分析的曲目进行分组，因此仅大小写或 Unicode 规范化形式不同的路径可能会合并到同一组。

例如，`Artist/Album/01 Song.flac` 会显示在 `Artist/Album` 子文件夹组中，也会在实际的 **文件夹** 树中依次位于 `Artist` 和 `Album` 下。不同索引根目录下相同的相对路径仍会分别显示。直接存放在根目录中的文件不会创建子文件夹组，但仍可在 **曲目** 以及 **文件夹** 中该根目录的直接曲目列表中使用。

使用 **搜索音乐库** 可以跨曲目、专辑、艺人和播放列表搜索。PC 布局中，可以从曲目列表表头按标题、艺人、专辑、流派和时长排序。专辑、艺人、流派、子文件夹和播放列表视图提供由目录数据库支持的 **排序** 列表。根据当前视图，可以按名称、艺人、年代、路径、曲目数、总时长、更新时间或创建时间进行升序或降序排列。每个视图会分别保留自己的选择。

搜索曲目时，三个或更多字符的关键词可匹配标题、艺人、专辑、流派、文件名或路径中的任意位置。一个或两个字符的关键词只匹配词语开头。如需匹配词语中间，请至少输入三个字符。

在 PC 和移动布局中，当曲目搜索结果、**树状视图** 当前层级的直接曲目，或专辑、艺人、流派、子文件夹、播放列表详情中的曲目不超过 300 首时，所有曲目都会默认选中；达到 301 首时则不会自动选择。可使用行复选框、**全选** 或 **取消全选** 更改选择状态。

移动端首先显示普通的曲目标题列表，不显示艺人列和时长列。只有长按曲目才会进入选择模式并显示复选框、**全选** 和 **取消全选**，同时保留常用的行操作。自动选择、**全选**、**取消全选** 和单个复选框操作只会改变选择状态，不会进入或退出选择模式。

如果没有元数据或无法读取元数据，EffeTune会使用文件名和文件夹信息显示。曲目属性中可以查看文件路径、格式、采样率、位深度、比特率和主要元数据项目。对于CUE曲目，还会显示曲目类型、CUE文件路径、源音频路径，以及它在源文件中的播放区间。

## CUE专辑文件

请将外部`.cue`文件与其中指定的WAV或FLAC文件放在同一文件夹，然后添加或重新扫描该文件夹。每个有效的`TRACK ... AUDIO`条目都会在音乐库中显示为一首独立曲目。CUE中有标题、表演者、日期、流派和曲目编号时会优先使用这些信息；音频技术信息则来自源WAV或FLAC文件。

对于已添加到音乐库的曲目，EffeTune会优先使用源音频文件中的内嵌封面。如果没有内嵌封面，则按顺序在CUE文件所在的文件夹中查找`cover.jpg`、`cover.png`、`front.jpg`、`front.png`，然后查找与源音频文件同名的JPEG或PNG（文件名可以包含或不包含音频扩展名）。在桌面应用中直接播放时，会自动使用相同的相邻图片候选；此播放路径不会从源音频文件中提取内嵌封面。在浏览器中直接播放时，则使用可从所选文件或已注册文件夹访问的相应图片。

你也可以通过 **Open music files** 直接播放CUE专辑；在移动设备上，请使用 **Open Music**。桌面应用还可以使用 **File > Open music file...**，只需选择`.cue`文件。在电脑端Chromium浏览器中，请先把专辑文件夹添加到音乐库并允许访问。之后只需选择`.cue`文件，EffeTune就会从已登记的文件夹中打开其引用的WAV或FLAC文件和匹配的封面，而不会把此次选择加入目录。不支持File System Access的浏览器仍需同时提供一个`.cue`文件、其中引用的全部且仅限这些WAV或FLAC文件，以及需要使用的匹配封面。有效选择会替换当前播放队列。验证失败时，当前队列不会改变。

如果CUE表无效，或无法安全确定它引用的源文件，EffeTune会说明问题，并把WAV或FLAC作为整文件的普通曲目导入。请修正CUE表或文件名，然后重新扫描文件夹。

## 从音乐库播放

选择曲目、专辑、艺人、流派、子文件夹、文件夹、搜索结果或播放列表，然后使用以下操作。

- **播放** - 替换当前播放器队列并开始播放。
- **随机播放** - 以随机顺序播放所选曲目组。
- **下一首播放** - 将所选曲目插入到当前曲目之后。
- **添加到队列** - 将所选曲目添加到队列末尾。
- **添加到播放列表** - 将所选曲目保存到音乐库的播放列表中。

在 PC 上，可以双击曲目行从该位置开始播放，也可以右键点击或通过 **更多** 菜单打开曲目操作。在移动端，点按普通列表中的曲目即可播放；长按会进入上述选择模式。

普通音乐播放器的控制和循环/随机播放设置仍然可用。可使用键盘的设备上，也可以使用普通播放器键盘快捷键。如果文件夹离线，无法打开音乐库中的曲目，请重新连接或重新导入该文件夹。

## 更新和重新连接文件夹

在音乐文件夹中添加、删除、重命名文件或编辑标签后，请使用 **重新扫描**。重新扫描会更新已更改的曲目，将找不到的文件从目录中移除，并尝试重新解析之前不可用的播放列表项目。

可以在 **文件夹** 画面的状态显示中确认该文件夹是否可用。

- **正常** - 文件夹可用。
- **未扫描** - 尚未建立索引。
- **未找到** - 文件夹或保存的路径不可用。
- **重新连接** - EffeTune 需要您重新授予访问权限。

如果文件夹显示为 **重新连接**，请选择 **重新连接**，并重新允许访问同一个文件夹。移除文件夹只会将它从音乐库目录中移除，不会删除磁盘上的文件。

## 播放列表

音乐库的播放列表保存在 EffeTune 内，可以包含已索引文件夹中的曲目。

可以执行以下操作。

- 从所选的音乐库曲目创建播放列表。
- 使用 **将队列保存为播放列表** 保存当前播放器队列。
- 对播放列表进行重命名、复制、删除和重新排序。
- 拖动播放列表中的曲目来调整顺序。在不便拖动的环境中，使用 **上移**、**下移**。
- 使用 **导入播放列表** 导入 M3U、M3U8、PLS、XSPF 格式的播放列表。
- 打开某个具体的播放列表，然后使用 **导出 M3U8** 或 **导出 XSPF** 将其导出。

### 最近播放与收藏

EffeTune 会在同一个卡片网格中，将两个特殊播放列表与普通播放列表并列显示。它们只在需要时创建：开始播放已索引曲目时创建 **最近播放**，首次用星标标记曲目时创建 **收藏**。

- **最近播放** 按从新到旧的顺序保留最近 100 首不重复的曲目。再次播放某首曲目时，它会移回列表顶部。
- **收藏** 包含用 ☆ 标记的曲目。在 PC 上，请使用曲目旁的星标；在移动端，请打开曲目的 **更多** 菜单。在 PC 上右键点击曲目也可以打开同一菜单。

这些播放列表的名称固定，并以当前界面语言显示，因此无法重命名。你仍可以像处理普通播放列表一样复制、导出或删除它们。删除后，下次播放记录或收藏操作需要该列表时，系统会以空列表重新创建。卡片的封面区域会显示时钟或星形图标；点击 **收藏** 卡片右下角的播放按钮即可立即播放。特殊播放列表不包含在普通播放列表的搜索结果中。

扫描文件夹时，EffeTune 会在曲目索引完成后自动导入支持的播放列表文件，并跳过内容未改变的文件。如果同一文件夹和相对路径下的文件内容发生变化，EffeTune 会以原子方式替换自动导入播放列表中的项目；在 EffeTune 中对这些项目所做的编辑也会被替换。导入失败或取消后，会在下次重新扫描时重试。删除源文件或为其重命名不会删除现有播放列表，重命名后的文件会作为新的播放列表导入。

导入时，会预览当前音乐库中匹配到的项目数量。未匹配的项目也会尽可能作为未解析项目保留，因此之后添加或重新连接相应文件夹时可以解析。

导出时选择 **相对路径** 后，如果可行，会导出相对于导出位置的路径。需要将播放列表和音乐文件夹一起移动时，这很有用。M3U8和XSPF无法保留CUE曲目在专辑文件中的播放区间，因此EffeTune会跳过这些曲目并显示跳过数量。导出内容绝不会用专辑文件的实际路径来代替被跳过的CUE曲目。

## 安全性和保存位置

- 音乐库会读取音乐文件和元数据，但不会向音乐文件写入更改。
- 封面缓存和播放列表是应用内数据，不是写入音乐文件的嵌入式更改。
- 子文件夹组根据目录中已保存的相对路径生成。
- 浏览器的存储区域可能会因浏览器设置或用户操作而被清除。重要播放列表请按需导出。
- 在支持 File System Access 的浏览器中，重新加载后能否复用持久保存的文件夹句柄取决于访问权限。使用回退方式时，所选文件仅在当前会话中有效，每次重新加载后都必须重新选择。

## 大型音乐库

目录将数据保存在磁盘上，并以分页或有界批次的方式处理，因此无需一次性把大型收藏全部载入内存。规模测量和固定基准测量只是可选的本地开发诊断，并非 commit、release、`verify` 或 GitHub Actions 的门槛，也不构成一般性的性能保证。扫描时间和实际容量取决于存储速度、可用内存、元数据、封面以及浏览器或操作系统的限制。

滚动曲目列表时，EffeTune 会缓存当前位置附近的页面。在移动端布局中，它会沿当前滚动方向预读最多两页，优先处理屏幕上需要的页面，并复用显示范围重叠的曲目行。即使滚动仍在继续，已完成的视口读取也会立即发布到这个有界缓存中。显示位置请求会合并到最新位置；如果刚完成加载的页面已经包含该位置，就不会再次读取数据库。已不再需要的待处理预读会被丢弃。SQLite 本身支持中断，但当前的目录适配器会同步执行每条语句，也没有提供从另一个 Worker 发出中断的路径。因此，在存储设备较慢时，如果以极快速度跳到较远位置，仍可能短暂显示空白，直到当前读取完成。
